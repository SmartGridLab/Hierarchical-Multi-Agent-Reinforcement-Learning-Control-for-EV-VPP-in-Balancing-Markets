================================================================
Hierarchical MARL control Data Folder Guide
================================================================
Last updated: 2026-04-27

----------------------------------------------------------------
[Overview]
This folder contains the input datasets used for training and evaluation.
They are grouped into three main categories:
  input.demand_fromPJM/ ... demand-adjustment data used during training
  input_EVinfo/         ... EV profile and arrival-related files
----------------------------------------------------------------


================================================================
input.demand_fromPJM/
================================================================

"8 2024.xlsx" to "12 2024.xlsx"   approx. 35-39 MB each
  Description: Monthly PJM-side demand-adjustment source files for August through December 2024.
  Usage: Converted into daily CSV files in `output_5min/` by `5mindivide.py`.
         The generated CSVs are used directly by training and evaluation.
  Related files:
    - data/input.demand_fromPJM/5mindivide.py
    - tools/execute.py (`--xlsx` option for custom input)
  Notes: The current `output_5min/` set contains 62 daily files from 2024-08-01 to 2024-12-31.

"5mindivide.py"   approx. 3 KB
  Description: Splits a monthly Excel file into one daily 5-minute CSV per day.
  Usage: Preprocessing script for generating `output_5min/`.
  Referenced by: tools/execute.py

"output_5min/"   approx. 248 KB (62 files: `day_2024-08-01.csv` to `day_2024-12-31.csv`)
  Description: Daily demand-adjustment CSV files generated by `5mindivide.py`.
               Each file corresponds to one day and contains 288 five-minute steps.
  Usage:
    - Main demand input for training and evaluation
    - Loaded by `readcsv.py`
    - Passed into `EVEnv` as `net_demand_series`
  Referenced by:
    - Config.py (`DEMAND_ADJUSTMENT_DIR`)
    - environment/readcsv.py (`load_multiple_demand_files`)
    - training/train.py (`sample_episode_demand_strict`)
  Notes: Values are min-max scaled in `readcsv.py` from an assumed raw range of `[-500, +1000]` kW.


================================================================
input_EVinfo/
================================================================

"WORKPLACE_neededsocanddwelltime.csv"   approx. 69 KB
  Description: EV profile table containing dwell time and target SoC information.
               Built from ACN-Data-derived workplace charging sessions and expanded into a 10,000-EV profile pool.
  Usage:
    - Main input file for EV profile sampling
    - Used when the environment samples departure timing and target SoC for incoming EVs
  Referenced by:
    - Config.py (`EV_PROFILE_DATA_PATH`)
    - environment/ev_info_loader.py

"soc_arrival_distribution.csv"   approx. 1.2 KB
  Description: Histogram-style CSV describing the SoC distribution at EV arrival.
  Usage: Used when sampling the initial SoC of arriving EVs.
  Referenced by:
    - Config.py (`EV_SOC_ARRIVAL_DISTRIBUTION_PATH`)
    - environment/ev_info_loader.py

"arrivals_by_station/"   approx. 200 KB (8 files)
  Description: Per-station EV arrival-rate CSV files indexed by day-of-week and time-of-day.
               Generated from `Electric_Vehicle_Charging_Station_Data_*.csv` via `build_arrival_profiles_by_station.py`.
  Usage: Used by `EVEnv` to sample station-specific arrival events.
  Referenced by:
    - Config.py (`PER_STATION_ARRIVAL_PROFILE_PATHS`)
    - environment/EVEnv.py (`_pregenerate_arrival_events`)
  Notes: A subset of five station files is currently selected in `Config.py`.

"Electric_Vehicle_Charging_Station_Data_-8671638762898357044.csv"   approx. 22 MB
  Description: Raw ACN-Data charging-station session dataset used to build `arrivals_by_station/`.
  Usage: Only needed when regenerating arrival profiles.
  Referenced by:
    - input_EVinfo/build_arrival_profiles_by_station.py

"transactions.csv"   approx. 158 KB
  Description: Raw EV charging transaction/session data.
  Usage: Used for generating arrival-profile CSV files such as `Arrival.csv`.
         Not used directly during normal training runs.
  Referenced by:
    - input_EVinfo/build_arrival_profile_from_transactions.py

"Arrival.csv"   approx. 4.2 KB
  Description: Aggregated arrival-rate CSV.
  Usage: Legacy/generated helper output kept for reference.

--- Auxiliary preprocessing scripts ---

"arrivesoc.py"   approx. 6 KB
  Description: Script for creating `soc_arrival_distribution.csv`.

"neededsoc_and_dwell.py"   approx. 5.3 KB
  Description: Script for creating `WORKPLACE_neededsocanddwelltime.csv`.

"build_arrival_profiles_by_station.py"   approx. 4.7 KB
  Description: Script for generating per-station arrival profiles in `arrivals_by_station/`.

"build_arrival_profile_from_transactions.py"   approx. 3.7 KB
  Description: Script for building arrival-related CSVs from `transactions.csv`.
